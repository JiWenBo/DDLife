# DDBook 产品设计方案

> “好的产品是用完即走的，但留下的记忆是永恒的。”

## 1. 产品愿景
这是一个连接孩子与书本的“记忆盒子”，旨在记录 6 岁孩子的阅读轨迹，并通过声音留存珍贵的成长瞬间。区别于市面上复杂的打卡工具，本 APP 强调**极简交互**与**情感连接**。

## 2. 页面架构与导航 (Navigation Architecture)

### 2.0 底部导航栏 (Bottom Tab Bar)
APP 采用经典的底部导航结构，分为三个核心 Tab，兼顾孩子的使用便捷性和家长的管理需求。默认进入中间的 **“读书”** Tab。

| Tab 位置 | 图标 (Icon) | 名称 | 核心功能 | 面向用户 |
| :--- | :--- | :--- | :--- | :--- |
| **左 (Left)** | 📖 `menu_book` | **书架 (Library)** | 展示所有绘本，家长管理入口 | 孩子 + 家长 |
| **中 (Center)** | 📷 `qr_code_scanner` | **读书 (Read)** | **(默认页)** 极简扫码入口，呼吸动画 | 孩子 |
| **右 (Right)** | 📊 `bar_chart` | **成长 (Growth)** | 阅读热力图，阅读习惯分析 | 家长 |

---

### 2.1 极简首页 (The Eye) - 中间 Tab
*   **设计理念**：所见即所得，打开 APP 唯一的动作就是“开始阅读”。
*   **功能点**：
    *   **呼吸扫描键**：屏幕中央一个律动的圆形按钮，点击唤起摄像头。
    *   **手动补录入口**：在扫描键下方提供一个低调的文字按钮“找不到条码？手动记录”，点击弹出书名搜索框，支持模糊匹配书架已有的书籍。
    *   **智能识别**：扫描绘本封底 ISBN 条码，秒级识别书籍信息（封面、书名、作者）。
    *   **每日签到**：无需手动打卡，扫码或手动记录成功即视为“今日阅读”。
    *   **彩蛋反馈**：
        *   新书首次阅读：屏幕绽放烟花或长出嫩芽。
        *   重读旧书：显示“这是你第 N 次读这本书啦！”。

### 2.2 声音胶囊 (The Voice)
*   **设计理念**：6 岁的声音稍纵即逝，这是比阅读量更珍贵的资产。
*   **功能点**：
    *   **朗读时刻**：扫码后，自动弹出麦克风图标，邀请孩子朗读一段最喜欢的段落（限时 60 秒）。
    *   **自动归档**：录音文件自动绑定到该书的“阅读历史”中。
    *   **时光回溯**：在书架页面，点击书籍可播放历史录音，听到 6 岁、7 岁、8 岁不同的声音变化。

### 2.3 智能书架 (The Library) UI 布局设计
*   **设计理念**：极致克制的文字列表流。在没有图片封面干扰的情况下，通过排版、留白、色彩点缀和层次感，打造一个清晰、高效、高级的“纯文本书目”。

*   **页面结构设计**：
    *   **1. 顶部导航栏 (AppBar)**：
        *   左侧：大标题“我的书架”与当前藏书总数（如：我的书架 · 共 128 本）。
        *   右侧：**核心入口区**。包含“搜索”图标和最显眼的 **“+”号（录入书籍入口）**。
    *   **2. 核心控制区 (Filter) - 位于 AppBar 下方**：
        *   **快捷标签栏 (Chips)**：横向滑动的标签，用于快速筛选。整合了阅读状态和书籍分类。例如：`[全部]` `[未读]` `[已读]` `[绘本故事]` `[自然科普]` 等。
    *   **3. 主体展示区 (纯文本列表视图 List View)**：
        *   **基础条目 (ListTile) 布局**：
            *   **左侧视觉锚点**：取消封面展示，左侧用一个**圆形的单字 Avatar** 代替。取书名的第一个字，背景色根据书名的 Hash 值生成固定的柔和色系（如浅红、浅绿、浅蓝），文字颜色为同色系深色。
            *   **主标题**：书名（字号稍大，加粗），如果书名很长则换行或截断。
            *   **副标题（信息行）**：字号较小，颜色较浅。显示 `分类 · 套系 · 册数`，例如：“自然科普 · 《神奇校车》 · 第3册”。
            *   **标签行**：书名下方展示灰色胶囊样式的 Tag（如 `#睡前故事`）。
            *   **右侧尾部**：
                *   如果已读，显示绿色的 `[✔ N次]` 徽标。
                *   包含一个“更多 (⋮)”按钮，点击弹出操作菜单（目前支持删除）。
        *   **套系书的特殊处理（折叠列表）**：
            *   当连续多本书属于同一个“套系”时（例如《神奇校车》1-10册），列表不直接平铺10行。
            *   显示为一条**“套系聚合条目”**：左侧图标为一个“文件夹”或“堆叠”样式，主标题为“套系：《神奇校车》”，右侧显示“共 10 册”。
            *   点击该条目，**向下展开** (ExpansionTile) 具体的10本书的细明细列表。这样可以在纯文字模式下极大节省屏幕空间，保持清爽。
    *   **4. 底部悬浮条 (可选，视交互需要)**：
        *   当长按某本书进入“多选模式”时，底部弹出操作条：[批量删除] [批量打标签] [加入淘汰箱]。

### 2.4 家长书库与录入功能 (The Manager & Entry)
*   **设计理念**：数据录入是家长的任务，阅读体验是孩子的任务。家长负责“进货”，孩子负责“消化”。录入过程必须追求极致的效率与兼容性。
*   **入口**：书架页面右上角的“+”号或隐藏的“管理模式”。
*   **核心录入功能设计**：
    *   **1. 扫码辅助录入流程**：
        *   **交互逻辑**：点击录入 -> 扫码获取 ISBN（或直接手动输入） -> 进入信息填写表单 -> 自动填入 ISBN 字段 -> 家长手动填写其余信息 -> 点击保存 -> 写入本地存储。
        *   *注：考虑到没有完美的免费公共 API，扫码动作仅作为“免去手动输入 13 位数字”的便捷工具，所有书籍信息（书名、分类等）由家长手动控制，确保数据准确性。*
    *   **2. 录入字段设计（极简与实用并重）**：
        *   **核心必填/自动获取**：
            *   **ISBN 码**（扫码自动获取/防重核心）
            *   **书名**（必填）
        *   **选填字段（完善信息与童书特色）**：
            *   **封面图**（拍照或本地选图，非必填）
            *   **书的类型/分类**（如：绘本、科普、英语、分级阅读等）
            *   **第几套/套系**（如：《大侦探恐龙》）
            *   **第几册**（如：第 3 册，与套系配合解决买重/漏买问题）
            *   **第几版**（如：第1版，供家长精准备注）
            *   **适读年龄**（如 3-6岁，方便后期筛选）
            *   **音频/点读包关联**（是否有配套的小雅/毛毛虫点读音频路径）
        *   **管理维度字段（系统自动生成或辅助）**：
            *   **录入时间**（系统自动生成，用于统计“藏书增长”）
            *   **状态标签**（已读/未读/愿望单/淘汰箱）
        
        *   **录入数据模型 (JSON 示例)**：
            ```json
        {
          "id": "1696123456789",
          "isbn": "9787533259563",
          "title": "猜猜我有多爱你",
          "author": null,
          "coverUrl": null,
          "category": "绘本故事",
          "seriesName": "信谊世界精选图画书",
          "volumeNumber": 1,
          "edition": "朋友赠送",
          "audioPath": null,
          "tags": ["睡前故事", "爱表达"],
          "readCount": 0,
          "lastReadAt": null,
          "createdAt": "2023-10-01T10:00:00.000Z"
        }
        ```
    *   **2. 童趣分类与灵活标签**：
        *   提供适合 6-12 岁的默认分类（如：绘本故事、自然科普、历史人文、少儿文学、英语分级、漫画/桥梁书、拼音认读、其他）。
        *   支持自由添加标签（Tag），如 `#睡前故事`、`#培养专注力`、`#需要讲解`，便于日后通过标签快速筛选。

    *   **3. 弱化非核心信息，强化套系管理**：
        *   **作者**：字段保留但不展示在录入页面，减轻家长录入负担。
        *   **封面**：非必填（当前版本纯文字无封面展示）。
        *   **套系与版次**：提供“套系名称”（如：大侦探恐龙）和“第几册”（如：3）两个独立字段，并提供单独一行“版次/备注”（如：第1版，或 朋友赠送）字段，方便家长管理成套书籍。
        *   **适读年龄、愿望单**：移除（默认所有录入书籍均为已购入库）。

    *   **4. 高效连续录入**：
        *   保存成功后，仅通过 SnackBar 提示“录入成功”，不跳回上一页。
        *   清空 ISBN、书名、册数等核心内容，但保留分类、标签等选项，方便家长连续录入同一系列或同一分类的多本书。
        *   **双重保存入口**：除了右上角的保存按钮，在表单最底部（备注下方）增加一个全宽的“保存书籍”大按钮，方便填完表单后直接保存，无需回滚页面。

    *   **5. 数据导出与备份 (Data Export)**：
        *   **导出为 JSON**：考虑到未来可能需要将数据迁移到云端服务器或其他平台，提供一键导出所有录入书籍和阅读记录为 `books.json` 文件的功能。家长可以通过微信、AirDrop 或邮件发送给自己。

## 3. 数据分析与成长 (The Growth)

### 3.1 核心数据结构设计 (Reading Record Model)
为了支持时间维度的热力图回顾和每日阅读明细，系统引入独立的**“阅读流水表” (Reading Log)**。与书籍信息表 (Book) 分离，专注于记录“阅读行为”本身。

*   **设计原则**：Book 表只记录当前状态（如总阅读次数、最后阅读时间），而 ReadingRecord 表记录每一次阅读的历史轨迹。
*   **数据模型 (ReadingRecord JSON 示例)**：
    ```json
    {
      "id": "1696123456789_bookId123",  // 唯一记录 ID
      "bookId": "bookId123",             // 关联的书籍 ID (外键)
      "readAt": "2023-10-01T20:30:00.000Z", // 阅读时间 (精确到秒)
      "durationSeconds": 1200,           // 阅读时长 (可选，单位：秒)
      "note": "孩子非常喜欢这一页"          // 心得/备注 (可选)
    }
    ```

### 3.2 阅读热力图与历史回溯 (Heatmap & History)
*   **设计理念**：将枯燥的数据转化为可视化的成长隐喻。
*   **展现形式**：
    *   **日历热力图 (Calendar Heatmap)**：
        *   **逻辑**：遍历所有 `ReadingRecord`，按 `readAt` 的日期 (yyyy-MM-dd) 进行分组统计。
        *   **视觉**：使用“脚印”或色块深浅代表当日阅读量（例如：0本=灰色，1-2本=浅绿，3-5本=中绿，>5本=深绿）。
        *   **交互**：点击热力图上的某一天（如 10月1日），下方展示当日阅读明细。
    *   **每日阅读明细 (Daily List)**：
        *   **逻辑**：根据选中的日期，查询 `ReadingRecord` 表中该日期的所有记录，并通过 `bookId` 关联查询 `Book` 表获取书名和详情。
        *   **展示**：以时间轴或列表形式展示当天读过的所有书。

### 3.3 习惯分析 (The Insight)
*   **阅读偏好雷达图**：分析孩子阅读的书籍类型（如：文学 40%、科学 30%、艺术 20%）。
*   **阅读时段分布**：记录孩子通常在什么时间阅读（晨读、睡前故事）。
*   **高频词云**：从书名中提取高频词（如“恐龙”、“太空”、“朋友”），生成孩子的年度关键词。

## 4. AI 赋能：解决父母痛点的智能体验 (AI Features)

在完善了基础的书籍管理和打卡记录后，引入 AI 能力可以极大提升 DDBook 的实用性和趣味性，直击父母在育儿阅读中的核心痛点。以下是规划的 AI 赋能场景：

### 4.1 痛点：买书容易，找书难 / 容易买重
**场景**：“上次买的那套《神奇校车》第二辑放到哪里去了？”、“双十一凑单，结果发现家里已经有一本《猜猜我有多爱你》了。”

*   **智能扫码防重（已具备基础，可 AI 增强）**：
    *   在书店或电商平台买书前，直接用首页扫码。AI 不仅提示“家里已有”，还能基于系列名分析：“这本书属于《xxx》套系，你已经有第1、2册了，这是第3册，可以买！”
*   **AI 家庭图书馆定位器**：
    *   录入书籍时，支持自然语言输入位置（如语音说：“这本放在客厅左边的书架最下层”），AI 自动解析并存储位置标签。找书时可以直接问：“AI，我的《霸王龙》放在哪了？”

### 4.2 痛点：不知道该给孩子读什么（选书焦虑）
**场景**：“家里的书都读了几遍了，接下来该买什么书？”、“这周要出游，带哪几本书合适？”

*   **AI 智能书单推荐（基于行为分析）**：
    *   AI 分析 `reading_records`（阅读记录）和 `books`（书籍标签/分类）。如果发现孩子最近频繁打卡“恐龙”标签，AI 会在成长页面提示：“您的孩子正处于**自然探索敏感期**”，并自动推荐同类经典绘本。
*   **“今晚读什么” AI 摇号器 / 阅读盲盒**：
    *   不知道睡前读什么时，点击“摇号”。AI 根据“很久没读（`lastReadAt`）”、“孩子最喜欢的分类”以及“当前时间（如睡前适合温和的故事）”，智能挑选出一本最合适的书，并给出推荐理由：“这本书你已经 2 个月没读了，今晚重温一下吧！”

### 4.3 痛点：讲故事能力有限 / 孩子总要求重复讲
**场景**：爸妈加班晚，爷爷奶奶带娃不会讲绘本；或者孩子要求同一本书连续讲 10 遍，家长嗓子冒烟。

*   **AI 绘本角色扮演（Voice Clone & TTS）**：
    *   **核心亮点**：家长只需朗读一段 1 分钟的样音，AI 提取声纹特征。之后，遇到任何书架上的书（通过网络匹配文本），APP 都能**用爸爸或妈妈的声音**，甚至模仿绘本中不同角色的语气（大灰狼粗犷、小兔子尖细）给孩子讲故事。
*   **AI 互动提问（阅读理解伴读）**：
    *   打卡完成后，AI 根据书名自动生成 1-2 个简单的互动问题（例如读完《猜猜我有多爱你》，AI 问：“宝宝，大兔子是怎么表达爱的呀？”），引导家长和孩子进行高质量对话，而不是读完就合上书。

### 4.4 痛点：二手书流转与闲置浪费
**场景**：“孩子长大了，3岁看的低幼绘本现在完全不看了，占地方又舍不得扔。”

*   **AI 一键生成“闲置漂流海报”**：
    *   AI 自动筛选出 `readCount` 为 0 或 `lastReadAt` 超过 2 年的书籍。
    *   家长一键勾选，AI 自动生成优美的转让文案（如：“这是一套陪伴我家宝宝度过 3 岁启蒙期的好书，九成新，寻找有缘的小主人”）和排版精美的海报，方便发到小区群或闲鱼。

### 4.5 痛点：缺乏有效的阅读正反馈
**场景**：孩子阅读有时候像完成任务，家长也只是口头表扬，缺乏长期的激励机制。

*   **AI 阅读里程碑与定制绘本**：
    *   当孩子读完 50 本书时，AI 提取孩子读过的书名中的高频词（如“恐龙”、“太空”），**自动生成一个专属于孩子的短篇小故事**，甚至可以调用 AI 绘画生成一张专属插图，作为里程碑奖励。
*   **AI 年度阅读报告**：
    *   年底时，AI 像网易云音乐一样生成年度报告：“这一年，你最爱在晚上 8 点阅读，你是一个‘小小探险家’（因为读了最多科普书）...”，极具情感价值。

## 5. 数据表结构设计 (Database Schema)

为了未来平滑地接入网络端（例如基于 RESTful API、GraphQL 或 Supabase/Firebase 等 BaaS 服务），当前本地 JSON 存储的数据结构已按照标准的关系型/文档型数据库表结构进行了设计。

主要包含两张核心表：**书籍表 (`books`)** 和 **阅读记录流水表 (`reading_records`)**。

### 5.1 书籍表 (`books`)
用于存储每一本绘本的元数据以及该书当前的汇总状态。
*在关系型数据库中，主键为 `id`。*

| 字段名 (Field) | 数据类型 (Type) | 必填 | 说明 (Description) | 示例值 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | String (UUID/Snowflake) | 是 | 书籍唯一标识符 | `"1696123456789"` |
| `isbn` | String | 是 | 国际标准书号（扫码获取） | `"9787533259563"` |
| `title` | String | 是 | 书名 | `"猜猜我有多爱你"` |
| `author` | String | 否 | 作者（当前前端弱化输入） | `null` |
| `coverUrl` | String | 否 | 封面图网络地址或本地路径 | `null` |
| `category` | String | 否 | 书籍分类（童趣分类） | `"绘本故事"` |
| `seriesName` | String | 否 | 套系名称（用于聚合） | `"信谊世界精选"` |
| `volumeNumber` | Integer | 否 | 册数（用于套系内排序） | `1` |
| `edition` | String | 否 | 版本/备注信息 | `"朋友赠送"` |
| `audioPath` | String | 否 | 配套音频/点读包路径 | `null` |
| `tags` | Array/JSONB | 否 | 标签集合 | `["睡前故事", "爱表达"]` |
| `readCount` | Integer | 是 | **总阅读次数（打卡累加器）** | `0` |
| `lastReadAt` | DateTime (ISO 8601) | 否 | 最后一次打卡的时间 | `"2023-10-01T10:00:00.000Z"` |
| `createdAt` | DateTime (ISO 8601) | 是 | 录入书架的时间 | `"2023-10-01T09:00:00.000Z"` |

### 5.2 阅读记录流水表 (`reading_records`)
用于存储每一次“打卡”动作的详细时间戳，支持成长页面的热力图渲染和历史回溯。
*在关系型数据库中，主键为 `id`，`bookId` 为指向 `books` 表的外键。*

| 字段名 (Field) | 数据类型 (Type) | 必填 | 说明 (Description) | 示例值 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | String (UUID) | 是 | 记录唯一标识符 | `"rec_1696123456789"` |
| `bookId` | String | 是 | **关联的书籍 ID (外键)** | `"1696123456789"` |
| `readAt` | DateTime (ISO 8601) | 是 | 具体的打卡/阅读时间 | `"2023-10-01T20:30:00.000Z"` |
| `durationSeconds`| Integer | 否 | 阅读时长（秒，为未来计时功能预留）| `1200` |
| `note` | String (Text) | 否 | 读后感/打卡备注 | `"孩子非常喜欢这一页"` |

## 6. 技术可行性分析 (Flutter)

*   **扫码识别**：使用 `mobile_scanner` 库，结合 OpenLibrary/豆瓣/Google Books API 获取元数据。
*   **本地存储**：使用 `Isar` 数据库，高性能存储海量书籍和阅读记录，支持离线使用。
*   **录音功能**：使用 `flutter_sound` 或 `record` 库，支持 AAC 格式录音，体积小音质好。
*   **图表绘制**：使用 `fl_chart` 绘制雷达图和趋势图，热力图可自定义 CustomPainter 实现。
*   **UI 框架**：Material 3 设计规范，适配 iOS 和 Android。

## 6. 开发路线图 (Roadmap)

1.  **Phase 1 (MVP)**：扫码识别 + 本地书架 + 基础阅读记录。

3.  **Phase 3 (Data)**：阅读热力图与习惯分析报表。
4.  **Phase 4 (Polish)**：UI 动效优化与导出功能。
